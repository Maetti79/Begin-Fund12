Executive Summary

This report documents the successful implementation of BeginPay, a modern Cardano (ADA) payment gateway plugin for WooCommerce, fulfilling the requirements of Catalyst Fund 12 Milestone 3. The plugin enables any WooCommerce merchant to accept ADA payments with CIP-30 wallet integration, real-time fiat-to-ADA conversion, QR code fallback, and on-chain payment verification via Blockfrost.

Implemented Features

1. CIP-30 Wallet Connection

Full CIP-30 dApp connector integration supporting 11 Cardano wallet extensions including Nami, Eternl, Lace, Flint, Typhon, GeroWallet, Yoroi, Vespr, Begin, and NuFi. Automatic wallet detection, one-click connection, and visual wallet status display.

Complete

2. In-Browser Transaction Building

Lightweight CBOR transaction builder (~10KB) that constructs valid Cardano Babbage-era transactions directly in the browser. Replaces the need for the 27MB Cardano Serialization Library while producing byte-identical output. Handles UTXO selection, fee estimation, change output, and TTL calculation.

Complete

3. Real-Time ADA/Fiat Conversion

Live exchange rate fetching from CoinMarketCap API with support for all WooCommerce currencies. Cart total is automatically converted to ADA with a one-click rate refresh button. Exchange rate and ADA amount are persisted to order metadata for accounting.

Complete

4. QR Code & Copy Address Fallback

Three-tab payment interface offering Connect Wallet, QR Code scan, and Copy Address methods. Copy-to-clipboard for both merchant address and ADA amount with visual feedback. QR code configurable via admin settings.

Complete

5. On-Chain Payment Verification

Blockfrost API integration for real-time on-chain transaction verification. Automatic polling checks transaction status, confirmation count, and verifies the correct ADA amount was sent to the merchant address. Order status auto-updates from "on-hold" to "processing" upon confirmation.

Complete

6. HPOS-Compatible Order Management

Full compatibility with WooCommerce High-Performance Order Storage. All order metadata (TX ID, ADA amount, exchange rate, payment method, verification status) stored via HPOS-safe methods. Admin order view, thank-you page, and email notifications include Cardano payment details.

CBOR Transaction Builder

A custom lightweight CBOR encoder that builds valid Cardano Babbage-era transactions without external dependencies. The output is byte-for-byte identical to transactions produced by the official Cardano Serialization Library (CSL v11.5.0).

  1. Retrieve UTXOs from the connected wallet via CIP-30 getUtxos()
  2. Parse each UTXO's CBOR to extract transaction hash, index, and lovelace amount
  3. Select UTXOs using a greedy algorithm (largest-first) until payment + fee + min-UTXO is covered
  4. Decode the merchant's bech32 address to raw hex bytes using an inline bech32 decoder
  5. Build the transaction body as a CBOR map: inputs (key 0), outputs (key 1), fee (key 2), TTL (key 3)
  6. Construct the full Transaction: [body, empty_witness_set, is_valid=true, null_auxiliary]
  7. Call signTx(cbor_hex, partialSign=true) on the wallet API
  8. Handle both full-transaction and witness-set-only responses from signTx
  9. Submit the signed transaction via submitTx() and capture the TX hash

Blockfrost Verification Flow

  1. After TX submission (or manual TX ID entry), polling begins every 10 seconds
  2. AJAX endpoint calls Blockfrost GET /txs/{hash} to check transaction existence
  3. If found, fetches GET /blocks/latest to calculate confirmation count
  4. Calls GET /txs/{hash}/utxos to verify the merchant address received the correct ADA amount
  5. Once confirmed (≥1 confirmation), the WooCommerce order status transitions from "on-hold" to "processing"
  6. TX ID, ADA amount, exchange rate, and confirmation count are saved to order metadata
  7. Checkout form auto-submits after on-chain confirmation, redirecting to the thank-you page

Security Model

🔒
WordPress Nonce Verification

All AJAX endpoints are protected by WordPress nonce verification (check_ajax_referer) to prevent CSRF attacks. Nonces are generated server-side and passed to the frontend via wp_localize_script.

🔓
On-Chain Verification

Payments are verified directly on the Cardano blockchain via Blockfrost API. The plugin checks that the correct amount of ADA was sent to the merchant's address by inspecting transaction UTXOs, preventing spoofed or insufficient payments.

🛡
Client-Side Key Isolation

Private keys never leave the user's wallet. The plugin uses the CIP-30 signTx API which delegates signing to the wallet extension. The merchant server never has access to customer wallet credentials.

✅
Input Sanitization

All user inputs (TX hashes, order IDs, amounts) are sanitized using WordPress core functions (sanitize_text_field, absint) and validated with strict regex patterns (64-character hex for TX hashes).

💰
Order Status Protection

Order confirmation AJAX validates that the order belongs to the BeginPay payment method and is in "on-hold" status before allowing status transitions. Already-processed orders cannot be re-confirmed.
